# Exercice Intermédiaire 01 — Mock réseau

## Énoncé

Vérifier l'affichage d'un catalogue alimenté par API mockée.

## Critères d'acceptation

1. Intercepter `/api/products`.
2. Afficher 3 produits mockés.
3. Tester le total des prix affichés.

---

## 🟢 Checkpoint — Auto-évaluation

Avant de passer à l’exercice suivant, vérifie que tu sais :

- Intercepter une requête réseau avec Playwright.
- Injecter des données mockées dans un test.
- Vérifier une logique métier côté front.

**Questions rapides**

1. Pourquoi mocker l’API dans un test E2E ?
2. Comment s’assurer que le mock est bien utilisé ?
3. Que tester en plus du total affiché ?
